System Architecture

A multi-model AI agent system for autonomous quantitative research

📱 Voice / Text Command (Telegram) │ ▼ 🤖 Claude Opus 4 — Primary Agent / Orchestrator │ ├──── 🖥️ Research Server 1 (112 threads, 754GB RAM) │ └── Kaiko: 64 exchanges, tick-level OB + trades (10yr) │ ├──── 🖥️ Research Server 2 (via SSH) │ └── TAQ · WRDS (272 DBs) · NASDAQ · Numerator │ ├──── ☁️ Cloud Server — Data collection + automation │ ├──── 🧠 Gemini 3.1 Pro — Paper writing, deep reasoning ├──── ⚡ Gemini Flash — Audio/video transcription (10x Whisper) │ ├──── 📊 30+ Financial Data APIs │ ├── Crypto: Binance · Hyperliquid · Polymarket · DefiLlama │ ├── Equities: Yahoo Finance · Twelve Data · Alpha Vantage │ ├── Macro: FRED · World Bank · ECB │ └── Options: Deribit · CBOE │ └──── 📝 Stateful Memory System (error logs, lessons learned)

Data Layer

Kaiko Consolidated Data

10 years of tick-level order book snapshots (up to 5,000 bid/ask levels) and raw trades across 64 cryptocurrency exchanges. Spot, perpetual, quarterly futures, and options.

TAQ (Trade and Quote)

NYSE TAQ millisecond-resolution data for all US-listed equities. Intraday microstructure, price impact estimation, cross-market event studies.

WRDS Academic Databases

Direct PostgreSQL to 272 databases: CRSP, Compustat, IBES, Thomson Reuters, RavenPack, and more. Full spectrum of financial and economic data.

Real-Time Market Data

Live connections to Binance, Hyperliquid (REST + WebSocket + S3), Polymarket (Gamma + CLOB), and 25+ additional providers across all asset classes.

AI Model Layer

Claude Opus 4 — Orchestrator

Manages the full research lifecycle: data exploration → hypothesis → code → analysis → figures → paper → iteration. Stateful memory across sessions.

Gemini 3.1 Pro — Deep Reasoning

Long-form academic writing, complex analytical reasoning, cross-model review. Publication-quality prose with proper econometric methodology.

Gemini Flash — Multimodal

Audio transcription (10x Whisper for Chinese-English), video analysis, rapid summarization. Native audio generation for research reports.

Stateful Experience System

Structured error log (错题本) prevents repeated mistakes. Technical state persists; research judgments start fresh to avoid confirmation bias.

Design Philosophy

Stability Over Speed

Run at 80% capacity, never 100%. Prevent OOM crashes and data corruption. Every analysis script is independently runnable with built-in error handling.

Stateful for Technical, Stateless for Research

Technical details (file paths, field names, bugs) accumulate in a structured log. Research judgments start fresh — past conclusions don't constrain new hypotheses. Prevents confirmation bias while avoiding repeated mistakes.

Honest Replication

All discrepancies between original papers and our replications are documented transparently. When results differ from the literature, the difference is reported, not hidden.

Global Compute Leverage

Each API call rents access to infrastructure worth billions — Google's TPU clusters, Anthropic's GPU farms, exchange matching engines. The research budget buys a ~20,000,000× cost-to-infrastructure leverage ratio.
